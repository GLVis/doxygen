18 [GLVis](https://glvis.org) is an OpenGL tool for visualization of finite element
19 meshes and functions. It is a multiplatform application that can be built on
20 Linux/Unix systems, including macOS, and under Windows. It can also be used in a
21 Jupyter notebook, or in a web browser, see https://glvis.org/live.
22 
23 - For building instructions, see [INSTALL](INSTALL).
24 
25 - The GLVis [key commands](#key-commands) and [mouse functions](#mouse-functions)
26  are documented below.
27 
28 - GLVis is distributed under the terms of the BSD-3 license. All new contributions
29  must be made under this license. See [LICENSE](LICENSE) and [NOTICE](NOTICE) for details.
30 
31 We welcome contributions and feedback from the community. Please see the file
32 [CONTRIBUTING.md](CONTRIBUTING.md) for additional details about our development
33 process.
34 
35 When started without any options, glvis starts a server which waits for a socket
36 connections (on port `19916` by default) and visualizes any received data. This
37 way the results of simulations on a remote (parallel) machine can be visualized
38 on the local user desktop.
39 
40 GLVis can also be used to visualize a mesh with or without a finite element
41 function (solution), as in
42 
43 ```
44 glvis -m cube.mesh3d
45 ```
46 
47 For parallel computations, GLVis supports input from several parallel socket
48 connections as well as the visualization of parallel meshes and grid functions
49 saved in separate files from the command line as in
50 
51 ```
52 glvis -np 4 -m mesh -g solution
53 ```
54 
55 When given parallel input, GLVis will stitch the results to show the global mesh
56 and solution. GLVis can also run a batch sequence of commands (GLVis scripts),
57 or display previously saved socket streams.
58 
59 For a complete list of command line options, type
60 
61 ```
62 glvis -h
63 ```
64 
65 Depending on the data type, variety of manipulations can be performed by using
66 the mouse and by typing (case sensitive) keystrokes in the GLVis window. Below
67 is a partial list of the available functionality. Some of these keys can also be
68 provided as input, using the `-k` command-line option and the `keys` script
69 command.

99 Key commands
100 ============
101 
102 ## Basic
103 
104 - <kbd>h</kbd> – Print a short help message in the terminal
105 - <kbd>r</kbd> – Reset the plot to 3D view
106 - <kbd>R</kbd> – Cycle through the six 2D projections (camera looking above/below in `x`/`y`/`z` directions)
107 - <kbd>j</kbd> – Turn on/off perspective
108 - <kbd>s</kbd> – Turn on/off unit cube scaling
109 - <kbd>A</kbd> – Turn on/off the use of anti-aliasing/multi-sampling
110 - <kbd>L</kbd> – Turn on/off logarithmic scale
111 - <kbd>c</kbd> – Toggle the colorbar and caption display state
112 - <kbd>C</kbd> – Change the main plot caption
113 - <kbd>p</kbd> / <kbd>P</kbd> – Cycle forward/backwards through color palettes (lots of options, use <kbd>F6</kbd> for a menu)
114 - <kbd>t</kbd> – Cycle materials and lights (5 states)
115 - <kbd>i</kbd> – Toggle cutting plane (different options in 2D and 3D, see below)
116 - <kbd>o</kbd> / <kbd>O</kbd> – Control element subdivisions (different options in 2D and 3D, see below)
117 - <kbd>l</kbd> – Turn on/off the light
118 - <kbd>g</kbd> – Toggle background color (white/black)
119 - <kbd>a</kbd> – Toggle the bounding box *axes*. The options are:
120  - none
121  - bounding box with coordinates of the corners
122  - bounding box without coordinates
123  - red, green, blue colored main `x`, `y`, `z` axes + dashed axes
124 - <kbd>m</kbd> – Toggle the *mesh* state. The options are:
125  - no mesh or level lines
126  - draw the element edges (i.e. the mesh)
127  - draw the level lines (use <kbd>F5 </kbd> to modify the level lines)
128 - <kbd>e</kbd> – Toggle the *elements* state (see below for vector functions). The options are:
129  - show surface elements (corresponding to the function)
130  - show no surface elements
131  - (2D only) show element attributes
132  - (2D only) show element `det(J)`
133  - (2D only) show element `1/det(J)`
134  - (2D only) show element `\kappa`
135  - (2D only) show element `\kappa + 1/\kappa`
136 - <kbd>S</kbd> – Take an image snapshot or record a movie (in spinning mode). By
137  default, the screenshots are taken in `png` format, using libpng. When GLVis
138  is compiled with `libtiff` support (see [INSTALL](INSTALL)) then the
139  screenshots are taken internally and saved in TIFF format (`.tif`
140  extension). If both of these options are disabled during the build process,
141  GLVis will use `SDL` to take screenshots in `bmp` format, which it will then
142  convert to `png` if ImageMagick's `convert` tool is available.
143 - <kbd>G</kbd> – 3D scene export to [glTF format](https://www.khronos.org/gltf)
144 - <kbd>Ctrl</kbd> + <kbd>p</kbd> – Print to a PDF file using `gl2ps`. Other
145  vector formats (SVG, EPS) are also possible, but keep in mind that the
146  printing takes a while and the generated files are big.
147 - <kbd>q</kbd> – Exit

172  - none

175 - <kbd>~</kbd> - Enter new ruler origin
176 - <kbd>k</kbd> / <kbd>K</kbd> - Adjust the transparency level. The balance of

182 - <kbd>F6</kbd> – Palette menu (negative repeat number flips the palette)
183 - <kbd>F7</kbd> – Change the minimum and maximum values
184 - <kbd>Shift</kbd> + <kbd>F7</kbd> – Set the bounding box from the terminal

186 ## 2D scalar data
187 
188 - <kbd>i</kbd> – Toggle cutting (clipping) plane in 2D
189 - <kbd>y</kbd> / <kbd>Y</kbd> – Rotate cutting plane (`\theta`) in 2D
190 - <kbd>z</kbd> / <kbd>Z</kbd> – Translate cutting plane in 2D
191 - <kbd>o</kbd> / <kbd>O</kbd> – Control element subdivisions in 2D
192  - there are two subdivision factors in this case: element (`s1`) and boundary (`s2`).
193  - <kbd>O</kbd> cycles through the following "subdivision functions", (prints a message in the terminal when changed):
194  - Increase element subdivision factor: `s1 += s2`
195  - Decrease element subdivision factor: `s1 -= s2`
196  - Increase boundary subdivision factor: `s2++`
197  - Decrease boundary subdivision factor: `s2--`
198  - <kbd>o</kbd> – performs the curently selected function
199 - <kbd>b</kbd> – Toggle the boundary in 2D scalar mode. The options are:
200  - no boundary
201  - black boundary
202  - boundary colored with the boundary attribute
203  - Use <kbd>Shift</kbd> + <kbd>F9</kbd> / <kbd>F10</kbd> to cycle through the boundary attributes.
204 - <kbd>F3</kbd> / <kbd>F4</kbd> – Shrink/Zoom each element towards its center, in order to better visualize the different element shapes
205 - <kbd>F8</kbd> – List of material subdomains to show
206 - <kbd>F9</kbd> / <kbd>F10</kbd> – Walk through material subdomains
207 - <kbd>F11</kbd> / <kbd>F12</kbd> – Shrink/Zoom material subdomains (to the centers of the attributes)

223 - <kbd>o</kbd> / <kbd>O</kbd> – Refine/de-refine elements in 3D

233 ## 2D vector data
234 
235 - <kbd>v</kbd> – Toggle the *vector* state (uses vector subdivision factor, accept <kbd>u</kbd> / <kbd>U</kbd>). The options are:
236  - do not show vectors
237  - show vectors as displacement
238  - show vector field; vectors are uniformly scaled; the color varies with the
239  magnitude (or the current *vector-to-scalar function*, see keys <kbd>u</kbd> / <kbd>U</kbd>)
240  - show vector field as above, but the vectors are scaled proportionally to their magnitude
241 - <kbd>V</kbd> – Change the scaling of the vectors relative to the default
242 - <kbd>d</kbd> – Toggle the *displaced mesh* state: (see also keys <kbd>n</kbd> / <kbd>b</kbd>). The options are:
243  - do not show displaced mesh
244  - show displaced mesh
245  - assuming displacement field show deformation using Cartesian lines
246  - assuming displacement field show deformation using polar lines
247 - <kbd>n</kbd> – Increase the displacement amount in 10% steps, wraps around from 100% to 0%
248 - <kbd>b</kbd> – Decrease the displacement amount in 10% steps, wraps around from 0% to 100%
249 - <kbd>B</kbd> – Toggle the boundary in 2D vector mode
250 - <kbd>e</kbd> – Toggle the *elements* state (vector data version). The options are:
251  - show surface elements corresponding to the current *vector-to-scalar function*
252  - do not show surface elements
253  - assuming a displacement field show `det(J)/det(J_d)`
254  - assuming a displacement field show `det(J_d)/det(J)`
255 - <kbd>u</kbd> / <kbd>U</kbd> – Change the *vector-to-scalar function* and the vector subdivision factor
256 - <kbd>U</kbd> – Toggle the functionality of <kbd>u</kbd> (prints a message in the terminal when changed). The options are:
257  - Increase the vector subdivision factor
258  - Decrease the vector subdivision factor
259  - Cycle through *vector-to-scalar functions* choices:
260  - magnitude: `\sqrt{v_x^2+v_y^2}`
261  - direction from `-\pi` to `\pi`: `atan2(v_y,v_x)`
262  - `x`-component: `v_x`
263  - `y`-component: `v_y`
264  - divergence: `div(v)`
265  - curl: `curl(v)` [skipped for H(div) elements]
266  - anisotropy in `grad(v)` [skipped for H(div) elements]
