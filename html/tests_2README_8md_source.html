12 Automated testing for GLVis
13 ===========================
14 This directory contains scripts and data for regression testing of GLVis:
15 
16 - `glvis_driver.py` is a Python script for running test cases and performing
17  comparisons on generated screenshots. Currently, the only supported mode is
18  *stream-based testing*, where given a saved input stream, commands are added
19  to the end of the stream to generate a screenshot and exit. This screenshot
20  can then be compared against a baseline image.
21 
22 - Test streams and baselines used in automated testing are stored in another
23  repository, [GLVis/data](https://github.com/GLVis/data), which is linked
24  into this repository as a submodule under the directory data.
25 
26  To fetch the test data submodule, call `git submodule update --init` in an
27  existing clone of this repository.
28 
29 Dependencies
30 ------------
31 Running tests with the test driver requires an installation of Python, with the
32 `scikit-image` package. Install it using `pip` or your distribution's package
33 manager.
34 
35 CMake is required to run the automated tests as they are used in Github Actions.
36 The tests also expect `.png` images, so GLVis should be built with PNG support.
37 
38 CMake-based testing
39 -------------------
40 The test cases for automated testing are defined using `CTest`, and can be found
41 in the `CMakeLists.txt` file in this directory. This is the method that is used
42 by the Github Actions tests.
43 
44 To run all the tests:
45 - Ensure that you have pulled in the data submodule.
46 - Build GLVis using CMake, passing `-DENABLE_TESTS=ON` as a configuration
47  option.
48 - Run `ctest` or `make test` to run all the test cases.
49 
50 Note that a set of baselines need to be generated manually for your specific
51 system, due to differences in rendering between operating systems and graphics
52 hardware. After the first run of `ctest` on master, run `make rebaseline` in
53 the build directory. This will copy the generated images from the test run to a
54 system-specific baseline directory.
55 
56 Standalone testing
57 ------------------
58 You can also run an individual test case outside of `CTest`, with the following
59 command:
60 
61 ```sh
62 python3 glvis_driver.py -e [path to glvis] \
63  -s [path to stream file] \
64  -b [path to stream baseline *directory*] \
65  -a [additional commands to pass to glvis]
66 ```
67 
68 for example
69 
70 ```sh
71 python3 glvis_driver.py -e ../glvis -s data/streams/shaper.saved -b data/baselines/local -a "-lw 1 -mslw 1 -nohidpi"
72 ```
73 
74 A screenshot named "test.[stream file name].png" will be generated. If no path
75 to a stream baseline directory is given, the test will exit; otherwise it will
76 attempt to find and open the corresponding screenshot in the baseline directory
77 and compare the two images.
78 
79 Details: Screenshot-based testing
80 ---------------------------------
81 As implemented in the `glvis_driver.py` test,
82 [structural similarity (SSIM)](https://en.wikipedia.org/wiki/Structural_similarity)
83 is used to determine similarity between the generated test screenshot and the
84 baseline screenshot.
85 
86 The current cutoff for an image and its baseline is set to 0.999; this allows
87 for minor differences in rendering output.
88 
89 Details: Baselines
90 ------------------
91 Two sets of public baselines are maintained: `ubuntu-20.04` and `macos-10.15`,
92 each representing their corresponding Github Actions virtual environment. These
93 are selected based on the value of the `GLVIS_BASELINE_SYS` CMake variable (set
94 to `local` by default).
95 
96 You should leave this variable unset for development purposes, even if your
97 development platform shares the same operating system; baselines are usually
98 generated in these virtual environments with software rendering, and may not
99 match the output of your specific system/graphics hardware.
100 
101 Reminder: after running `ctest` on master, developers can run `make rebaseline`
102 to populate the local baseline with reference figures.
